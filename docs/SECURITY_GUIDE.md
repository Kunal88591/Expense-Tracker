# Security Guide: Handling Credentials Safely

## ⚠️ CRITICAL: Your Credentials Were Shared

If you shared MongoDB credentials in chat or on GitHub, **they are now compromised**. You must:

1. **Rotate credentials immediately** (5 minutes)
2. **Update local .env** (2 minutes)
3. **Re-deploy if live** (5 minutes)

---

## 🔒 How to Rotate MongoDB Credentials

### Step 1: Generate New Password in MongoDB Atlas

1. Go to https://www.mongodb.com/cloud/atlas
2. Login → Select your cluster
3. Go to **Security** tab → **Database Access**
4. Find user `kunalintern_db_user`
5. Click **Edit** (pencil icon)
6. Click **Edit Password**
7. Choose **Auto Generate Secure Password**
8. **Copy the new password** (only shown once!)
9. Click **Update User**

### Step 2: Get New Connection String

1. Still in MongoDB Atlas
2. Go to **Databases** → **Connect**
3. Choose **Drivers** → **Node.js**
4. Copy connection string: `mongodb+srv://kunalintern_db_user:NEW_PASSWORD@cluster0.feyvboy.mongodb.net/...`
5. Replace `NEW_PASSWORD` with the password you just generated

### Step 3: Update Local .env File

```bash
cd /workspaces/Expense-Tracker/backend

# Open .env editor
nano .env

# Update with new connection string
MONGODB_URI=mongodb+srv://kunalintern_db_user:YOUR_NEW_PASSWORD@cluster0.feyvboy.mongodb.net/?appName=Cluster0
JWT_SECRET=your-secret-key-minimum-32-characters-here
PORT=5000
FRONTEND_URL=http://localhost:3000
```

**Save:** Press Ctrl+O, then Enter, then Ctrl+X

### Step 4: Test New Credentials

```bash
cd /workspaces/Expense-Tracker/backend
npm run dev
```

You should see:
```
MongoDB Connected: cluster0.feyvboy.mongodb.net
Server running on port 5000
```

## ⚠️ If Already Deployed to Production

### For Render.com:
1. Go to your backend service
2. Click **Environment** tab
3. Update `MONGODB_URI` with new connection string
4. Click **Save**
5. Service auto-redeploys

### For Railway.app:
1. Go to MongoDB plugin → **Variables**
2. Check if auto-generated (usually safe)
3. If manual, update connection string
4. Redeploy backend service

### For Vercel Frontend:
1. No action needed (frontend doesn't store MongoDB credentials)
2. Just verify `REACT_APP_API_URL` points to backend

## ✅ Proper Credential Management

### What to DO ✅

1. **Keep .env local only**
   ```bash
   # .env is gitignored - safe locally
   MONGODB_URI=mongodb+srv://user:password@...
   JWT_SECRET=your-secret-key
   ```

2. **Use .env.example as template**
   ```bash
   # .env.example shows structure (safe to commit)
   MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/...
   JWT_SECRET=your-secret-key-min-32-characters
   ```

3. **Set secrets in deployment platforms**
   - Render: Environment → Variables
   - Railway: Variables dashboard
   - Vercel: Settings → Environment Variables

4. **Rotate credentials quarterly**

5. **Use strong passwords** (min 32 characters for both)

### What NOT to do ❌

1. **Never share credentials in chat/email**
2. **Never commit .env to git**
3. **Never hardcode secrets in code**
4. **Never use same password across services**
5. **Never log credentials to console in production**

## 🔑 Generating Strong Secrets

### JWT_SECRET (use one of these)

**Option 1: Using openssl**
```bash
openssl rand -base64 32
# Output: xyz123abc...
```

**Option 2: Using Node.js**
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

**Option 3: Online generator** (not recommended)
- Use only for local testing
- www.random.org (pick 32+ random characters)

### MongoDB Password (MongoDB generates for you)

1. MongoDB Atlas → Database Access → Edit User
2. Click **Edit Password**
3. Choose **Auto Generate Secure Password**
4. MongoDB creates strong password automatically

## 🔍 Checking If Credentials Are Safe

### Check .gitignore is working

```bash
cd /workspaces/Expense-Tracker

# Should be empty (file not tracked)
git status | grep .env
```

### Check .env not in git history

```bash
# Should return nothing
git log --all -- .env
```

### Check no credentials in code

```bash
# Search for MongoDB URI patterns
grep -r "mongodb+srv://" . --exclude-dir=.git
```

Should only find matches in `.env.example` (with placeholders).

## 📚 Reference

**Files to know:**
- `.env` - Your local secrets (gitignored)
- `.env.example` - Template (safe to commit)
- `backend/.env.example` - Backend template
- `frontend/.env.example` - Frontend template
- `.gitignore` - Specifies what git ignores

**Never commit these:**
- `.env`
- `.env.local`
- `.env.production`
- Anything with actual secrets
